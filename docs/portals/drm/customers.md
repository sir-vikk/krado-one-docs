# Customers

**Route:** `/drm/customers` · **Edition:** Community+

The customer registry lets you track which customers are affected by an incident. This is especially useful for generating customer communications and SLA breach reports.

![Customers](/assets/screenshots/drm/customers.png)

## Customer fields

| Field | Description |
|-------|-------------|
| Name | Customer or account name |
| Email | Primary contact email |
| Plan / Tier | Subscription tier (used for SLA prioritisation) |
| Notes | Internal notes about the customer |

## Linking to incidents

In the incident form, use the **Affected Customers** section to select one or more customers. The incident detail page will then show the customer impact count and allow you to track per-customer status.

## Search

Use the search bar at the top of the customers list to quickly find a customer by name or email.
